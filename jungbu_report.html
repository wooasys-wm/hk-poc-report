3. 비용 산출 방식

보고서 본문에는 두 가지 비용이 등장한다. 정의가 다르므로 반드시 분리한다.

3.1 실 운임 (실 비용 baseline)

원천 파일: app/db/jungbu_real_fare_by_shipment_nov_dec.csv. shipment 단위 운임 파일이며, 각 shipment 의 실제 톤급 / 실 거리 (dbs_km) 에 11월 단가표 bucket 을 적용한 fare_krw 가 기록되어 있다.

정의  실 비용 = shipment별 fare_krw 를 job_date 기준으로 합산한 일자별 총액.

3.2 vroom lookup 운임

원천 파일: app/db/fare_2025_11.csv + app/cost/fare_lookup.py. vroom 이 만든 각 route 의 거리를 단가표 bucket 에 다시 넣어 운임을 산출한다.

정의  vroom lookup 운임 = 각 route 거리 × 해당 톤급의 중부 11월 단가표 bucket fare 의 합.

표현 규칙
  • 실 비용 / 실 운임 = 실 shipment, 실 톤급, 실 dbs_km 기준
  • vroom 운임 / lookup 운임 = 최적화 route, vroom route 거리, route 톤급 기준

4.3 해석

  • As-Is 실 운임은 1T 중심 구조라 1T 총액 비중이 크다.
  • fare 는 큰 차 통합이 강해 8T / 11T 비중이 커지고 총 lookup 운임이 가장 낮다.
  • version_35_5 는 3.5T / 5T 비용 비중이 커진다.
  • version_1_35 는 1T 분산 운영으로 1T 비용 비중이 다시 커지고, 총 lookup 운임이 실 운임에 가장 가깝다.

5. 검증 방식

  • 기존 v12_2pass 의 11T hardcoding / priority / DELIVERY_OVERRIDES / cap / max_distance 룰 유지.
  • 변경한 것은 fleet 수와 산출 경로뿐.
  • 1차 (pass1) 는 B(Standard) fleet 로 27일 전 일자 replay.
  • 2차 (pass2) 는 B 안에서 미배차가 발생한 일자에만 C(Robust) fleet 로 full-day rerun, B 정상 일자 + C rerun 결과를 결합.
  • 판단 기준: 미배차 수 / 메저 100% 초과 route 수 / 병목 날짜 집중 여부.

1차 Peak fleet (B 의 1T 만 증차) 검증 결과 세 변형 모두 일부 일자에서 미배차가 남았다 (fare 10건, version_35_5 11건, version_1_35 2건). 미배차 일자의 공통 패턴은 3.5T + 5T + 8T + 11T 동시 만재. 8T(9) / 11T(4) 제약 하에서 Peak fleet 의 3.5T / 5T (필요 시 1T) 만 증차하여 재실행, 세 변형 모두 미배차 0 도달.

9. 결론

9.1 상시 fleet + Peak fleet 2단계 구조

미배차 0 을 만드는 단일 고정 fleet 를 상시 fleet 로 보는 것은 과도하다. 평상시를 담당하는 상시 fleet + 피크 일자를 흡수하는 Peak fleet 의 2단계 구조가 자연스럽다.

9.2 8T / 11T 제약 하에서 미배차 0 은 3.5T / 5T 증차로 흡수 가능

1차 Peak fleet 검증에서는 세 변형 모두 3.5T + 5T + 8T + 11T 동시 만재 패턴으로 미배차가 남았다. 8T(9) / 11T(4) 제약을 그대로 둔 채 3.5T / 5T (필요 시 1T) 증차로 세 변형 모두 0 미배차에 도달. 1차 대비 fleet 증가는 +3 ~ +5대.

  • 다른 두 변형은 메저 100% 초과 0건.
요약 문장   As-Is baseline 은 shipment 단위 실 운임을 날짜별로 합산한 3.15억이며, vroom lookup 운임은 route 거리 기준 단가표 bucket 운임이다. percentile 기반 fleet 후보안 (B(Standard) 상시 + C(Robust) Peak) 을 vroom replay 로 재검증한 결과, 1차에서는 세 변형 모두 Peak 일자에 3.5T + 5T + 8T + 11T 동시 만재 패턴으로 미배차가 남았다. 8T(9) / 11T(4) 를 고정한 채 Peak fleet 의 3.5T / 5T (필요 시 1T) 만 증차하여 세 변형 모두 미배차 0 에 도달하였다. 최종 운영안은 fleet 규모 순으로 fare 28 / 39대, version_35_5 27 / 39대, version_1_35 40 / 45대 (상시 / Peak).
